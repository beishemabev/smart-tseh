# CURRENT_STATE

**ОБЯЗАТЕЛЬНО ПРОЧИТАТЬ ПЕРЕД ЛЮБЫМ ЗАПРОСОМ.**

### Текущая Стадия:
**Готово к Production релизу на Vercel**. Версия 1.0.0 (MVP) полностью функциональна, типы проверены, код заморожен (Code Freeze). Проведена UI-полировка (Empty States) для плавного онбординга новых пользователей.

### Добавленные изменения:
- **PWA Конфигурация**: Настроен `next-pwa` (через `next.config.ts`), добавлен `manifest.json`, в `layout.tsx` прописаны мета-теги (`viewport-fit=cover`, `theme-color`, `standalone`).
- **Слой Данных (Гибридная Архитектура)**: Создан `src/lib/store.ts` (Zustand с persist middleware). Добавлены экшены CRUD для базовых сущностей и функция `restoreDatabase`. Подключен кастомный сторадж `idbStorage` для Offline-first работы (IndexedDB). Настроен клиент **Supabase** (`src/lib/supabase.ts`) для облачного бэкапа. Добавлена **Авто-синхронизация**: при логине пользователя (имя + телефон) стейт автоматически сохраняется в таблицу `workspaces` в фоновом режиме через `store.subscribe` с использованием debounce (debounce каждые 2 секунды).
- **Хуки**: Создан `useStoreHydration` для предотвращения ошибок гидратации Next.js при работе с локальными данными Zustand.
- **Утилиты**: Добавлены базовые хелперы в `src/lib/utils.ts` (`cn`, `generateId`, `formatCurrency`, `formatDate`, `downloadCSV`).
- **Страница Настроек**: Управление каталогом операций (`SettingsTaskList` и `SettingsTaskForm`). Блок "Облачная синхронизация" переведен на SaaS-модель (`CloudSyncManager.tsx`): форма логина для авторизации и восстановление из БД, если данные уже есть. Для авторизованных показан приветственный экран с индикацией фоновой синхронизации и возможностью логаута (очистки локальной БД).
- **Дашборд (Учет выработки)**: Создана главная страница `/page.tsx` с навигацией по датам (через `date-fns`). Написаны компоненты: `QuickEntryForm` (ввод сделанных единиц с фиксацией текущей цены), `DailyLog` (список операций за день с возможностью удаления), `DailySummary` (плавающая панель с итоговой суммой оплат и штук). Добавлена **Кнопка "Отправить итоги дня"** (экспорт дневных итогов в WhatsApp с подсчетом Лидера дня).
- **Быстрый Ввод (QuickEntryForm)**: Добавлены **Smart Defaults** (запоминание последнего выбранного сотрудника и операции), **Quick Tap Buttons** (кнопки +10, +30, +50 для быстрого добавления кол-ва) и **Auto-Sorting** (активные сегодня операции всплывают наверх в селекте).
- **Навигация**: Добавлен фиксированный снизу `BottomNav.tsx` для переключения между четырьмя основными экранами мобильного PWA (Учет, Швеи, Отчеты, Настройки).
- **Сотрудники (Швеи)**: Управление персоналом. Добавлена функция выдачи авансов: кнопка "Выдать аванс" в карточке сотрудника открывает модалку формы `AdvanceForm`. Авансы сохраняются в отдельный массив стора. Добавлен **Бейджик баланса** (автоматический расчет баланса 'заработано минус авансы' за текущую неделю).
- **Отчеты по зарплате**: Создана страница `/reports/page.tsx`. Логика фильтрации `workEntries` и `advances` по сотруднику и периоду дат. Вычет авансов из общей суммы заработка. Агрегирование суммы деталей и общей ЗП. Генерация чека для отправки в WhatsApp с учетом выданных авансов. Добавлена функция **экспорта в Excel (CSV)** (`downloadCSV` в `utils.ts`) для бухгалтерии, формирующая файл с подробной детализацией пошитых единиц и выданных авансов за период работает строго в offline режиме.

### Глобальные типы (`src/types/index.ts`):
- `Role`: ('seamstress' | 'manager' | 'owner')
- `User`: (id, name, role, phone?, createdAt)
- `TaskType`: (id, name, pricePerUnit, createdAt)
- `WorkEntry`: (id, date: YYYY-MM-DD, userId, taskId, quantity, priceAtTheTime, total, createdAt)
- `Advance`: (id, date: YYYY-MM-DD, userId, amount, reason, createdAt)
